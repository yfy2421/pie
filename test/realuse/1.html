1. EpayConfig 与 SiteConfig 存在并发读写竞争（数据竞争）

  • 位置：admin_handler.go:52 (UpdateEpayConfig) 写 + EpayNotify/GetPaymentInfo 读
  • 现状：EpayConfigLock 是 *sync.RWMutex — 指针类型。SavePayConfig 中：
    EpayConfigLock.Lock()
EpayConfig = config  // 换指针
EpayConfigLock.Unlock()
    但 LoadPayConfig() 也在用 EpayConfigLock.Lock()，且 GetEpayConfig() 读锁保护的是指针读取。然而在 http_handler.go:35 的 EpayNotify 中：
    epayConfig := config.GetEpayConfig()

    当指针被换掉后，旧指针可能被 GC，但读锁外的字段访问仍在继续。

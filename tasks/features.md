# 功能开发任务 ✨

## 原则

- **窄入口，深挖**——每个功能先做 80% 场景，不做完美
- **搜索和 Git 只做到"有"**——不是与 VS Code 竞争，是避免面板空占位
- **Agent 功能优先于编辑器功能**——记住项目叫 My Code Agent，不是 My Code Editor

## 任务

| 优先级 | 任务 | 说明 | 工作量 |
| --- | --- | --- | --- |
| P0 | **文件搜索 Pane** | 后端 `/api/search` + 前端 `pane/search/index.ts`。文件名搜索 + 全文搜索 + 高亮 + 文件图标 | ✅ 1.5 天 |
| P0 | **Git Pane（只读）** | 后端 `/api/git/status` + `/api/git/log`，前端变更列表+提交历史，着色状态徽章，文件点击打开 | ✅ 1.5 天 |
| P1 | **Monaco 语言特性** | tsserver 子进程（fork + IPC）提供 TS/JS 诊断、补全、悬停、定义、引用 | ✅ 2 天 |
| P1 | **聊天快捷功能** | 预设 prompt 模板（/explain、/refactor、/test）、快速模型切换 | ✅ 1 天 |
| P1 | **Token 用量小组件** | 从 PI SDK `getContextUsage()` 获取 token 数，对话输入框左侧实时显示 | ✅ 0.5 天 |
| P1 | **标签页/会话持久化** | 重启后恢复最后打开的代码文件标签或对话会话 | ✅ 0.5 天 |
| P1 | **引用文件** | 输入框 + 按钮选择文件/文件夹作为 LLM 上下文附件（参考 Claude Code `@` 引用） | 2 天 |
| P1 | **对话模式选择** | 解释/计划/自动三种模式 + Effort 滑动条档位选择（参考 Claude Code `/mode`） | 2 天 |
| P2 | **Git Pane（写入）** | commit message 输入 + commit 按钮 + push/pull | ✅ 2 天 |
| P2 | **设置页完善** | 通用设置（字体大小、缩进、主题色）实际功能，不止占位符文案 | ✅ 1 天 |
| P3 | **搜索增强** | .gitignore 感知 + 内容 grep + 结果分组 | 2 天 |
| P3 | **LSP 多语言架构** | 可插拔语言服务器系统，每种语言可单独启用/禁用/替换 LSP 服务器 | 3 天 |
